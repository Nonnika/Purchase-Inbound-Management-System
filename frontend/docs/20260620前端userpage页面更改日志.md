# 20260620 前端 UserPage 页面更改日志

## 概述

新增的三个接口，前端用户管理页实现了对应的搜索、删除、创建功能，构建通过并已提交（仅 `frontend/`，未 push）。

## 接口映射

| 后端接口 | 前端 API 方法 | 页面功能 |
|---------|--------------|---------|
| `GET /users/selectByUserName?username=` | `usersApi.selectByUserName` | 顶部搜索框，按用户名查找单个用户，未找到显示提示 |
| `GET /users/deleteById?id=` | `usersApi.deleteById` | 每行「删除」按钮，`window.confirm` 确认后调用并刷新列表 |
| `POST /users/insert` | `usersApi.insert` | 「新增用户」按钮打开 Modal 表单提交 |

## 改动文件

### API / 类型层

- `src/types/user.ts` — 新增 `UserInput`（insert 载荷：`username/password_hash/real_name/phone/role_id/department_id`）与 `AffectedResult`
- `src/api/users.ts` — 新增 `selectByUserName`、`deleteById`、`insert`，归一化后端 `{affected}` 响应

### 新增可复用组件

- `src/components/ui/TextInput/` — Carbon 下边框输入框（label + helper + error 态，0px 圆角、40px 高）
- `src/components/ui/Modal/` — 遮罩层对话框（scrim + 阴影面板，Esc/点遮罩关闭，`createPortal` 挂载）

### 页面

- `src/pages/users/UsersPage.tsx` — 工具栏（新增/刷新）、搜索栏、行内删除、创建 Modal 表单；抽出 `UserTable` 子组件复用列表与搜索结果渲染
- `src/pages/users/UsersPage.module.css` — 搜索框、操作列、表单双列布局样式

## 注意事项

### 1. 密码字段处理

后端 `dao.Insert` 直接把请求体的 `password_hash` 入库，并未调用已有的 `encode.EncodePasswd`（bcrypt）。因此表单的密码字段原样作为 `password_hash` 提交，UI 上加了 helper 文案说明「后端未在服务端做哈希」。

如果后续后端改为接收明文密码并在服务端哈希，前端把字段名/逻辑改一下即可。

### 2. Update 接口未暴露

DAO 里 `Update` 方法标注「不稳定」且未在 `RegisterRouter` 注册，所以前端暂未做编辑功能。等后端挂上更新路由后再补「编辑」入口。
